[ICON] 為什麼 Markdown 原本勝出？

簡單可讀

標題、列表、程式碼區塊與表格，足以支撐大多數 AI 文字輸出。

Token 省

早期 GPT-4 8,192 token 限制下，Markdown 比 HTML 更省上下文成本。

Git 友善

純文字 diff、版本控制、Obsidian / GitHub / docs pipeline 都成熟。

容易修改

模型容易生成，人類也能直接修，讓它成為 agent 工作流的共同語言。

Thariq 的反轉：AI 輸出變太複雜

Markdown 文字牆

  • 長規格容易掃過不讀
  • 多方案比較只能線性排列
  • 圖表常退化成 ASCII art
  • PR review 缺少視覺層級
  • 難做決策輸入與回傳
VS

HTML artifact

  • 可用版面引導閱讀
  • 並排、卡片、timeline、tabs
  • SVG、圖表、互動元件
  • diff annotation 與 severity color
  • button / slider / export 回 agent

反方觀點：HTML 不是免費午餐

Token 與成本

HTML 可能需要更多輸出 token，生成更慢，成本不可忽略。

Diff 與維護

大型 HTML 檔較難人工 review；Markdown 仍是長期紀錄的強項。

安全與信任

HTML 可能含 JavaScript；不可信檔案需要 sandbox 或審查。

輸出膨脹

如果 100 行 Markdown 沒人讀，可能也該先要求 agent 精簡。

最佳分工：Markdown for memory, HTML for review

Markdown 適合

  • 長期知識庫與 repo docs
  • AGENTS.md、README、規格紀錄
  • 版本控制與文字 diff
  • 人類與 agent 反覆編輯
  • 低成本中間素材
+

HTML 適合

  • 簡報、報告、教學頁
  • PR review 與設計審閱
  • 多方案比較與決策介面
  • 互動式調參與 micro-editor
  • 非技術同事瀏覽與分享
